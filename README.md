#### What I build

I work at the intersection of **machine learning, reinforcement learning, and financial markets**. My focus is the unsexy infrastructure that makes ML strategies actually survive contact with live trading — leak-proof validation, realistic backtests, and feature engineering grounded in the academic literature, not Twitter folklore.

When the abstractions are right, the same codebase generalises beyond finance — robotics control, supply-chain optimisation, recommender systems all reduce to sequential decision-making with the same env / agent / reward primitives.

#### Currently focused on

- 🚀 **[QuantFlow](https://github.com/vigilancetrent/quantflow)** — modular ML/RL library for financial markets. 20 modern indicators with citations, walk-forward CV by default, gym-compatible trading env, real backtest engine. MIT licensed.
- 🧠 Hybrid Transformer + RL strategies for short-horizon equity / FX
- 📊 Microstructure features (Kyle's λ, VPIN, fractional differentiation) and their predictive horizons
- 🔬 Distinguishing skill from luck in trading research — deflated Sharpe, multiple-testing correction, regime-conditioned policies

#### Principles

1. **The backtest is a lie** until it survives walk-forward CV with realistic costs.
2. **Citation or it didn't happen.** Every indicator I ship carries the original paper's equation.
3. **Trade the model, not the heroics.** Boring, well-validated alpha beats clever fragile alpha every time.
4. **Open source where possible.** Strategies are private; tooling shouldn't be.
